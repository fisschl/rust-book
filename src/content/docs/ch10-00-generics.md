title: 10.0. 概述

每种编程语言都有有效处理概念重复的工具。在 Rust 中，这样的工具之一是 _泛型_ ：具体类型或其他属性的抽象替身。我们可以表达泛型的行为或它们如何与其他泛型相关，而无需知道在编译和运行代码时什么将代替它们。

函数可以接受某些泛型类型的参数，而不是像 `i32` 或 `String` 这样的具体类型，就像它们接受未知值的参数以在多个具体值上运行相同的代码一样。事实上，我们已经在第 6 章中使用 `Option<T>`、第 8 章中使用 `Vec<T>` 和 `HashMap<K, V>` 以及第 9 章中使用 `Result<T, E>` 时使用了泛型。在本章中，你将探索如何定义自己的类型、函数和方法与泛型！

首先，我们将回顾如何提取函数以减少代码重复。然后我们将使用相同的技术从两个仅在参数类型上不同的函数中创建一个泛型函数。我们还将解释如何在结构体和枚举定义中使用泛型类型。

然后，你将学习如何使用 trait 以泛型方式定义行为。你可以将 trait 与泛型类型结合使用，将泛型类型限制为仅接受具有特定行为的类型，而不仅仅是任何类型。

最后，我们将讨论 _生命周期_ ：一种向编译器提供关于引用如何相互关联的信息的泛型。生命周期允许我们向编译器提供足够的关于借用值的信息，以便它可以在比我们不帮助它时更多的情况下确保引用有效。

## 通过提取函数消除重复

泛型允许我们用代表多个类型的占位符替换特定类型以消除代码重复。在深入了解泛型语法之前，让我们首先看看如何在不涉及泛型类型的情况下通过提取一个用代表多个值的占位符替换特定值的函数来消除重复。然后，我们将应用相同的技术来提取一个泛型函数！通过了解如何识别你可以提取到函数中的重复代码，你将开始识别可以使用泛型的重复代码。

我们将从代码示例 10-1 中的简短程序开始，该程序在列表中找到最大的数字。

**代码示例 10-1：在数字列表中找到最大的数字**

```rust
fn main() {
    let number_list = vec![34, 50, 25, 100, 65];

    let mut largest = &number_list[0];

    for number in &number_list {
        if number > largest {
            largest = number;
        }
    }

    println!("最大的数字是 {largest}");
}
```

我们在变量 `number_list` 中存储一个整数列表，并将对列表中第一个数字的引用放在名为 `largest` 的变量中。然后，我们遍历列表中的所有数字，如果当前数字大于存储在 `largest` 中的数字，我们就替换该变量中的引用。然而，如果当前数字小于或等于迄今为止看到的最大数字，该变量不会改变，代码继续处理列表中的下一个数字。在考虑列表中的所有数字后，`largest` 应该指向最大的数字，在本例中是 100。

现在我们的任务是在两个不同的数字列表中找到最大的数字。为此，我们可以选择复制代码示例 10-1 中的代码，并在程序中两个不同的地方使用相同的逻辑，如代码示例 10-2 所示。

**代码示例 10-2：在*两个*数字列表中找到最大数字的代码**

```rust
fn main() {
    let number_list = vec![34, 50, 25, 100, 65];

    let mut largest = &number_list[0];

    for number in &number_list {
        if number > largest {
            largest = number;
        }
    }

    println!("最大的数字是 {largest}");

    let number_list = vec![102, 34, 6000, 89, 54, 2, 43, 8];

    let mut largest = &number_list[0];

    for number in &number_list {
        if number > largest {
            largest = number;
        }
    }

    println!("最大的数字是 {largest}");
}
```

虽然这段代码有效，但复制代码是乏味且容易出错的。当我们想要更改它时，我们还必须记住在多个地方更新代码。

为了消除这种重复，我们将通过定义一个对作为参数传入的任何整数列表进行操作的函数来创建抽象。这个解决方案使我们的代码更清晰，并让我们抽象地表达在列表中找到最大数字的概念。

在代码示例 10-3 中，我们将找到最大数字的代码提取到一个名为 `largest` 的函数中。然后，我们调用该函数来找到代码示例 10-2 中两个列表中的最大数字。我们也可以在将来可能拥有的任何其他 `i32` 值列表上使用该函数。

**代码示例 10-3：在两个列表中找到最大数字的抽象代码**

```rust
fn largest(list: &[i32]) -> &i32 {
    let mut largest = &list[0];

    for item in list {
        if item > largest {
            largest = item;
        }
    }

    largest
}

fn main() {
    let number_list = vec![34, 50, 25, 100, 65];

    let result = largest(&number_list);
    println!("最大的数字是 {result}");

    let number_list = vec![102, 34, 6000, 89, 54, 2, 43, 8];

    let result = largest(&number_list);
    println!("最大的数字是 {result}");
}
```

`largest` 函数有一个名为 `list` 的参数，代表我们可能传入函数的任何具体的 `i32` 值切片。因此，当我们调用该函数时，代码在我们传入的具体值上运行。

总之，以下是我们将代码从代码示例 10-2 更改为代码示例 10-3 所采取的步骤：

1. 识别重复代码。
2. 将重复代码提取到函数体中，并在函数签名中指定该代码的输入和返回值。
3. 更新两处重复代码的实例以调用函数代替。

接下来，我们将使用这些相同的步骤与泛型来减少代码重复。就像函数体可以在抽象的 `list` 上操作而不是具体值一样，泛型允许代码在抽象类型上操作。

例如，假设我们有两个函数：一个找到 `i32` 值切片中的最大项，另一个找到 `char` 值切片中的最大项。我们如何消除这种重复？让我们找出答案！
