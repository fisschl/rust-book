---
title: 15.0. 概述
---

指针是一个通用的概念，它表示一个包含内存地址的变量。这个地址指向或“指向”其他数据。Rust 中最常见的指针类型是引用，你在第4章已经学习过。引用用 `&` 符号表示，借用它们指向的值。它们除了引用数据之外没有其他特殊能力，也没有额外的开销。

另一方面，_智能指针_是像指针一样行为的数据结构，但还包含额外的元数据和能力。智能指针的概念并非 Rust 独有：智能指针起源于 C++，也存在于其他语言中。Rust 在标准库中定义了各种智能指针，提供了超越引用的功能。为了探索这一通用概念，我们将研究几种不同的智能指针示例，包括一种 _引用计数_ 的智能指针类型。这种指针通过跟踪所有者的数量，让你可以让数据拥有多个所有者，并在没有所有者时清理数据。

在 Rust 中，基于所有权和借用的概念，引用和智能指针之间还有一个额外的区别：虽然引用只是借用数据，但在很多情况下智能指针 _拥有_ 它们指向的数据。

智能指针通常使用结构体实现。与普通结构体不同，智能指针实现了 `Deref` 和 `Drop` trait。`Deref` trait 允许智能指针结构体的实例像引用一样行为，这样你就可以编写能够与引用或智能指针一起工作的代码。`Drop` trait 允许你自定义当智能指针实例超出作用域时运行的代码。在本章中，我们将讨论这两个 trait 并演示它们对智能指针的重要性。

由于智能指针模式是 Rust 中经常使用的通用设计模式，本章不会涵盖所有现有的智能指针。许多库都有自己的智能指针，你甚至可以编写自己的智能指针。我们将涵盖标准库中最常见的智能指针：

- `Box<T>`，用于在堆上分配值
- `Rc<T>`，一种引用计数类型，支持多重所有权
- `Ref<T>` 和 `RefMut<T>`，通过 `RefCell<T>` 访问，该类型在运行时而不是编译时强制执行借用规则

此外，我们还将涵盖 _内部可变性_ 模式，即一个不可变类型公开一个用于改变内部值的 API。我们还将讨论引用循环：它们如何导致内存泄漏以及如何防止它们。
