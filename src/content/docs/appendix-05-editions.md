---
title: 22.5. E. 版本
---

在第 1 章中，你看到 `cargo new` 会向你的 *Cargo.toml* 文件添加一些关于版本的元数据。本附录将介绍这意味着什么！

Rust 语言和编译器有六周的发布周期，这意味着用户会获得源源不断的新功能。其他编程语言发布较大的更改频率较低；Rust 更频繁地发布较小的更新。一段时间后，所有这些微小的更改累积起来。但从一个版本到另一个版本，很难回顾并说："哇，在 Rust 1.10 和 Rust 1.31 之间，Rust 变化很大！"

每隔三年左右，Rust 团队会发布一个新的 Rust _版本_。每个版本将已经落地的功能整合成一个清晰的包，带有完全更新的文档和工具。新版本作为常规六周发布流程的一部分发布。

版本为不同的人服务不同的目的：

- 对于活跃的 Rust 用户，新版本将增量更改整合成一个易于理解的包。
- 对于非用户，新版本标志着一些重大进步已经落地，这可能使 Rust 值得再次关注。
- 对于开发 Rust 的人，新版本为整个项目提供了一个集结点。

在撰写本文时，有四个 Rust 版本可用：Rust 2015、Rust 2018、Rust 2021 和 Rust 2024。本书使用 Rust 2024 版本的惯用语编写。

*Cargo.toml* 中的 `edition` 键指示编译器应该为你的代码使用哪个版本。如果该键不存在，为了向后兼容，Rust 使用 `2015` 作为版本值。

每个项目可以选择加入默认 2015 版本以外的版本。版本可以包含不兼容的更改，例如包含与代码中标识符冲突的新关键字。但是，除非你选择加入这些更改，否则即使你升级使用的 Rust 编译器版本，你的代码也会继续编译。

所有 Rust 编译器版本都支持在该编译器发布之前存在的任何版本，并且它们可以将任何支持版本的 crate 链接在一起。版本更改仅影响编译器最初解析代码的方式。因此，如果你使用 Rust 2015，而你的某个依赖使用 Rust 2018，你的项目将编译并能够使用该依赖。相反的情况也适用，即你的项目使用 Rust 2018 而依赖使用 Rust 2015。

需要明确的是：大多数功能将在所有版本上可用。使用任何 Rust 版本的开发者将继续看到改进，因为新的稳定版本不断发布。然而，在某些情况下，主要是在添加新关键字时，一些新功能可能只在较新版本中可用。如果你想利用这些功能，你需要切换版本。

有关更多详细信息，请参见 [*The Rust Edition Guide*][edition-guide]。这是一本完整的书，列举了版本之间的差异，并解释了如何通过 `cargo fix` 自动升级你的代码到新版本。
