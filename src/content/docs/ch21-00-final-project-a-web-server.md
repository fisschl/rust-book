这是一段漫长的旅程，但我们已经走到了本书的末尾。在本章中，我们将再一起构建一个项目，来展示我们在最后几章中涵盖的一些概念，同时也回顾一些早期的课程。

对于我们的最终项目，我们将制作一个 Web 服务器，它会说 "Hello!"，在浏览器中看起来像图 21-1 所示。

以下是我们构建 Web 服务器的计划：

1. 学习一些关于 TCP 和 HTTP 的知识。
2. 在套接字上监听 TCP 连接。
3. 解析少量的 HTTP 请求。
4. 创建一个正确的 HTTP 响应。
5. 使用线程池提高服务器的吞吐量。

![浏览器访问 127.0.0.1:8080 的截图，显示一个网页，文本内容为 “Hello! Hi from Rust”](img/trpl21-01.png)

图 21-1：我们最终的共享项目

在我们开始之前，我们应该提到两个细节。首先，我们将使用的方法并不是用 Rust 构建 Web 服务器的最佳方式。社区成员已经在 [crates.io](https://crates.io/) 上发布了大量可用于生产环境的 crate，它们比我们即将构建的提供了更完整的 Web 服务器和线程池实现。然而，我们在本章的目的是帮助你学习，而不是走捷径。因为 Rust 是一种系统编程语言，我们可以选择我们想要工作的抽象层次，并且可以比其他语言更低级。

其次，我们不会在这里使用 async 和 await。构建一个线程池本身就是一个足够大的挑战，更不用说还要构建一个异步运行时了！然而，我们会注意到 async 和 await 可能如何适用于我们在本章中会遇到的一些问题。最终，正如我们在第 17 章中指出的，许多异步运行时使用线程池来管理它们的工作。

因此，我们将手动编写基本的 HTTP 服务器和线程池，以便你可以了解将来可能使用的 crate 背后的通用思想和技术。
