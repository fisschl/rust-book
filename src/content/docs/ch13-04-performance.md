---
title: 13.4. 循环与迭代器的性能
---

为了确定是使用循环还是迭代器，你需要知道哪种实现更快：使用显式 `for` 循环的 `search` 函数版本，还是使用迭代器的版本。

我们通过将阿瑟·柯南·道尔爵士的《福尔摩斯历险记》的全部内容加载到一个 `String` 中，并在内容中查找单词 _the_ 来进行基准测试。以下是使用 `for` 循环和使用迭代器的 `search` 版本的基准测试结果：

```text
test bench_search_for  ... bench:  19,620,300 ns/iter (+/- 915,700)
test bench_search_iter ... bench:  19,234,900 ns/iter (+/- 657,200)
```

这两种实现具有相似的性能！我们不会在这里解释基准测试代码，因为重点不是证明两个版本是等价的，而是大致了解这两种实现在性能上如何比较。

对于更全面的基准测试，你应该检查使用各种大小的文本来作为 `contents`，不同的单词和不同长度的单词作为 `query`，以及各种其他变化。重点是：迭代器虽然是一种高级抽象，但编译后与你自己编写的低级代码大致相同。迭代器是 Rust 的 _零成本抽象_ 之一，我们指的是使用这种抽象不会带来额外的运行时开销。这类似于 C++ 的原始设计者和实现者 Bjarne Stroustrup 在他2012年的 ETAPS 主题演讲"Foundations of C++"中对零开销的定义：

> 一般来说，C++ 实现遵循零开销原则：你不使用的东西，你就不用为之付费。更进一步：你使用的东西，你无法手动编写出更好的代码。

在许多情况下，使用迭代器的 Rust 代码会编译成与你手写的相同汇编代码。循环展开和消除数组访问边界检查等优化都会应用，并使结果代码极其高效。现在你知道了这一点，你可以毫无顾虑地使用迭代器和闭包！它们使代码看起来像是更高级的，但不会因此产生运行时性能损失。

## 总结

闭包和迭代器是受函数式编程语言思想启发的 Rust 特性。它们有助于 Rust 在低级别性能下清晰表达高级思想的能力。闭包和迭代器的实现使得运行时性能不受影响。这是 Rust 致力于提供零成本抽象的目标的一部分。

现在我们已经改进了 I/O 项目的表达能力，让我们看看 `cargo` 的一些更多特性，这些特性将帮助我们将项目分享给全世界。
